# Prompt-Engineering Grundlagen

Ein Sprachmodell ist so gut wie die Anweisung, die es bekommt. Vier Regeln decken 90 Prozent des Alltags ab.

## 1. Rolle und Ziel nennen

Statt „Fasse das zusammen": „Du schreibst für Lehrkräfte ohne Vorwissen. Fasse den Text in fünf Sätzen zusammen."

## 2. Format vorgeben

Sag explizit, was herauskommen soll: Tabelle, Stichpunkte, maximal 100 Wörter. Modelle halten sich erstaunlich gut an klare Formatvorgaben.

## 3. Beispiele statt Beschreibungen

Ein gutes Beispiel wirkt stärker als drei Absätze Erklärung. Aber Vorsicht: Manche Modelle schreiben Beispielwerte wörtlich ab — Platzhalter wie `<Name>` sind sicherer als erfundene konkrete Werte.

## 4. Iterieren, nicht resignieren

Der erste Prompt ist ein Entwurf. Was fehlt, wird zur nächsten Anweisung: „Kürzer. Ohne Fachbegriffe. Beispiele aus der Grundschule."

Verwandt: [[Lokale KI-Modelle]] — die Regeln gelten dort genauso, kleine Modelle brauchen sie sogar dringender.
